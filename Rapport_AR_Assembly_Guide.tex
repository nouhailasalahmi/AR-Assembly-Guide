\section{Introduction}
% =========================================================================
Dans le cadre de la transition vers l'industrie 4.0, l'optimisation des opérations d'assemblage mécanique sur site représente un défi majeur. La Réalité Augmentée (AR) offre une solution innovante en projetant des indications numériques et des repères tridimensionnels directement sur les pièces physiques. 

Ce projet présente le développement d'une application d'assistance à l'assemblage basée sur \textbf{Unity 6} et \textbf{Vuforia Engine}. L'application guide pas à pas un opérateur dans le montage d'un mécanisme à bielle, piston et vilebrequin. Elle propose un guidage interactif tridimensionnel, une validation automatique ou manuelle de la justesse de l'assemblage (tolérance de position et d'orientation) ainsi qu'un calcul de score de performance.

% =========================================================================
\section{Cahier des Charges et Objectifs}
% =========================================================================
L'application doit remplir les exigences fonctionnelles suivantes :
\begin{itemize}
    \item \textbf{Tracking Fiable} : Détecter et suivre précisément les composants physiques (le vilebrequin, la bielle, le piston) à l'aide de marqueurs d'images (\textit{Image Targets} Vuforia).
    \item \textbf{Guidage Étape-par-Étape} : Charger de manière dynamique la séquence d'instructions d'assemblage depuis un fichier de configuration standardisé au format JSON.
    \item \textbf{Validation en Temps Réel} : Analyser l'écart spatial (en mètres) et l'écart d'orientation (en degrés) entre les positions actuelles et cibles des pièces.
    \item \textbf{Modes de Validation Flexibles} :
    \begin{itemize}
        \item \textbf{Auto} : Validation automatique dès que la pièce est placée dans les tolérances définies.
        \item \textbf{Button} : L'opérateur clique manuellement sur un bouton pour vérifier son action.
        \item \textbf{Gesture} : Détection de gestes tactiles spécifiques sur l'écran.
    \end{itemize}
    \item \textbf{Évaluation de l'Opérateur} : Évaluation de la session (score final sur 100 points, nombre d'erreurs commises, temps total écoulé) et attribution d'une appréciation qualitative (Excellent, Bien, Correct, À améliorer, Insuffisant).
\end{itemize}

\subsection{La Machine à États (AssemblyStateMachine)}
La classe principale \texttt{AssemblyStateMachine} gère le cycle de vie de la session d'assemblage à travers les états définis par l'énumération \texttt{MachineState} :
\begin{itemize}
    \item \textbf{Idle} : En attente du démarrage par l'opérateur.
    \item \textbf{Running} : Étape en cours d'exécution.
    \item \textbf{Transitioning} : Phase intermédiaire de validation (durée courte avec retours visuels positifs).
    \item \textbf{Completed} : Fin de la procédure avec affichage du tableau de bord des résultats.
\end{itemize}

\subsection{Structure de Données de l'Étape (\texttt{AssemblyStep.cs})}
Chaque étape est désérialisée dans une instance de la classe \texttt{AssemblyStep} :

\begin{lstlisting}[caption=Classe AssemblyStep.cs, label=lst:step]
using UnityEngine;

[System.Serializable]
public class AssemblyStep
{
    public int id;
    public string instruction;
    public string annotationPrefabName;
    public Vector3 targetPosition;
    public Vector3 targetRotation;
    public float positionTolerance;
    public float rotationTolerance;
    public float maxDurationSeconds;
    public ValidationMode validationMode;
}

public enum ValidationMode
{
    Button,   // Bouton "Valider" sur l'UI
    Gesture,  // Gestes de balayage tactiles
    Auto      // Validation automatique par positionnement spatial
}
\end{lstlisting}

\subsection{Module de Validation (\texttt{ValidationModule.cs})}
Le script \texttt{ValidationModule} effectue les calculs géométriques en comparant la position et la rotation de l'objet suivi (\texttt{trackedObject}) par rapport aux cibles définies :

\begin{lstlisting}[caption=Algorithme de validation spatiale, label=lst:validation]
private bool CheckPositionAndRotation(AssemblyStep step, out string failReason)
{
    failReason = string.Empty;
    if (trackedObject == null)
    {
        failReason = "Objet non détecté par le tracking.";
        return false;
    }

    // 1. Vérification de la distance de position (en mètres)
    float posError = Vector3.Distance(trackedObject.position, step.targetPosition);
    if (posError > step.positionTolerance)
    {
        failReason = $"Position incorrecte (écart : {posError * 100f:F1} cm, tolérance : {step.positionTolerance * 100f:F0} cm).";
        return false;
    }

    // 2. Vérification de la rotation angulaire (en degrés)
    Quaternion targetRot = Quaternion.Euler(step.targetRotation);
    float rotError = Quaternion.Angle(trackedObject.rotation, targetRot);
    if (rotError > step.rotationTolerance)
    {
        failReason = $"Orientation incorrecte (écart : {rotError:F1}°, tolérance : {step.rotationTolerance:F0}°).";
        return false;
    }

    return true;
}
\end{lstlisting}

\subsection{Gestion du Score et Chronométrage (\texttt{ScoringManager.cs})}
Le système de notation évalue l'utilisateur selon deux critères : les erreurs commises et le temps nécessaire pour compléter chaque étape :

\begin{lstlisting}[caption=Calcul du score final, label=lst:scoring]
private int CalculateScore(float totalTime)
{
    int score = baseScore;

    // Déduction des points pour chaque erreur
    score -= _totalErrors * penaltyPerError;

    // Déduction en cas de dépassement du temps moyen alloué
    if (_totalSteps > 0)
    {
        float avgStepTime = totalTime / _totalSteps;
        float overtime = Mathf.Max(0f, avgStepTime - referenceStepDuration);
        int timePenalty = Mathf.FloorToInt(overtime / 30f) * penaltyPerTimeThreshold;
        score -= timePenalty;
    }

    return Mathf.Clamp(score, 0, 100);
}
\end{lstlisting}

\section{Manuel d'Installation et d'Utilisation}
% =========================================================================

\subsection{Configuration Matérielle et Logicielle requise}
\begin{itemize}
    \item \textbf{Logiciel} : Unity 6 (version recommandée : 6000.x) ou ultérieure.
    \item \textbf{SDK AR} : Vuforia Engine Package pour Unity.
    \item \textbf{Périphériques} : Webcam PC (pour tests dans l'éditeur) ou Smartphone Android/iOS avec caméra active.
\end{itemize}

\subsection{Déploiement et Test}
\begin{enumerate}
    \item Cloner le projet depuis GitHub :
    \begin{lstlisting}[language=bash]
    git clone https://github.com/nouhailasalahmi/AR-Assembly-Guide.git
    \end{lstlisting}
    \item Ouvrir le projet dans \textbf{Unity Hub} avec l'éditeur Unity 6.
    \item Imprimer les marqueurs cibles situés dans le dossier \texttt{Assets/Markers/}.
    \item Ouvrir la scène principale dans \texttt{Assets/Scenes/MainScene.unity}.
    \item Cliquer sur \textbf{Play}. Présenter les marqueurs devant la caméra pour tester le tracking et suivre les instructions à l'écran.
\end{enumerate}

\section{Conclusion et Perspectives}
% =========================================================================
L'application développée propose un flux de travail complet et interactif de guidage en Réalité Augmentée. Grâce à sa structure dynamique basée sur un fichier JSON de configuration et son système de validation spatiale adaptatif, elle offre une grande évolutivité. 

Les perspectives de développement futur incluent :
\begin{itemize}
    \item \textbf{Reconnaissance de Modèles 3D (CAD Model Tracking)} : Remplacer les marqueurs d'images 2D par la détection directe des modèles CAO des pièces réelles via la technologie de \textit{Model Target} de Vuforia.
    \item \textbf{Intégration d'un Système de Commande Vocale} : Permettre à l'opérateur de passer à l'étape suivante par commande vocale (ex. "Suivant", "Valider") afin de lui laisser les mains entièrement libres pour le montage physique.
    \item \textbf{Compatibilité Casque AR} : Porter l'application sur des casques de réalité mixte autonomes comme le Microsoft HoloLens ou le Meta Quest 3 pour une immersion totale.
\end{itemize}
